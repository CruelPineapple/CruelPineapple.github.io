关于JSX

JSX可以很好地描述UI的交互本质

看起来它是一个模板语言, 但它具有JavaScript的全部功能。

通过JSX可以生成React元素！

使用JSX的原因

React觉得渲染逻辑本质上与其他UI逻辑内在耦合(而我需要慢慢体会一下为什么React这样觉得)。总之就是UI的变化和数据在UI中的展示都是与它们对应的逻辑很有关系的。

想想写原生js是怎么做的，先形成页面，再在js中操作DOM，这就是所谓的人为分离了标记和逻辑的方式。

因此，React选择把标记和逻辑都放在一个耦合单元中，他们叫它组件。这实现了关注点分离

======

经过我绝赞的理解，关注点分离这个词实在有些抽象，通过例子来说明就容易理解很多。比如模块化，封装等，都是实现关注点分离的手段，这么看就比较清晰了，所谓关注点分离，就是把我们不需要的信息给藏起来，只展示我们关心的内容。而不是字面意义上理解的，给关注点分类。

======
